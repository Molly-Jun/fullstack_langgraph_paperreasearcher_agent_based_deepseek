# 论文摘要与问答笔记系统

一个围绕 PDF 论文阅读场景构建的全栈 Demo。用户可以上传论文、生成摘要、基于论文内容继续提问，并把每次问答自动整理为 Markdown 笔记。项目通过 LangGraph 组织摘要、问答与笔记三条工作流，同时在前端展示处理过程、摘要结果和问答结果，适合演示、学习和后续扩展。

## 项目简介

这个项目的目标不是做一个通用聊天机器人，而是做一个面向论文阅读与复盘的工作台。

用户上传 PDF 后，系统会解析论文内容，生成结构化摘要；随后用户可以围绕同一篇论文继续提问，系统会先生成答题计划，再在用户审批后输出答案与引用；同时，问答过程还会被保存为 Markdown 笔记，方便后续复习和整理。

项目的核心价值在于把“上传、阅读、摘要、提问、复盘”串成一条完整工作流，并把执行过程可视化出来。

## 特点

- **HITL 答题计划审批**：问答先生成计划，再由用户审批后继续执行。
- **问答结果带引用**：系统会返回基于论文内容的答案和引用锚点。
- **保存笔记**：每次问答都会沉淀为 Markdown 笔记。
- **工作流可视化**：前端会展示当前处理进度和活动时间线。
- **本地工作区管理**：上传的 PDF、摘要和笔记都会保存到本地，便于回看。

## 项目结构

- `frontend/`：前端页面，负责上传、展示摘要、问答与资源管理
- `backend/`：后端服务与 LangGraph 工作流
- `data/`：运行时生成的数据目录，包括 PDF、摘要和笔记

## 如何配置与启动

### 1. 环境要求

- Python 3.11+
- Node.js 18+（建议使用较新的 LTS 版本）
- 可用的 DeepSeek API Key

### 2. 配置后端环境变量

后端配置文件示例通常放在 `backend/.env.example`。你需要根据实际环境创建 `.env`，并填写必要变量。

建议至少配置：

- `DEEPSEEK_API_KEY`
- 如有需要，也可配置工作目录、模型名称等运行参数

### 3. 安装后端依赖

进入后端目录后安装依赖：

```bash
cd backend
pip install -e .
```

如果项目使用开发依赖，也可以按实际环境安装 dev 依赖。

### 4. 启动后端

在 `backend/` 目录下启动 FastAPI 服务。

```bash
langgraph dev
```
或者使用 `uvicorn` / 项目中已有的启动方式。

### 5. 安装前端依赖

进入前端目录后安装依赖：

```bash
cd frontend
npm install
```

### 6. 启动前端

前端开发模式启动：

```bash
npm run dev
```

启动后按照终端提示访问本地地址即可。

### 7. 打包与生产运行

如果需要构建前端静态资源：

```bash
npm run build
```

构建后可配合后端静态资源挂载或容器化方式运行。

## 原理说明

### 1. PDF 处理

上传 PDF 后，后端会先把文档解析为 Markdown，再按章节结构切分成可处理的文本块。这样既能保留论文结构，也方便后续摘要和问答使用。

### 2. 摘要流程

摘要链路大致分为四步：

1. 解析 PDF
2. 按章节切分并并行摘要
3. 反思当前摘要是否完整
4. 合并生成最终摘要并保存到本地

这种方式比一次性长文本生成更稳定，也更适合长论文。

### 3. 问答流程

问答链路采用了“先计划、后执行”的方式：

1. 根据用户问题生成答题计划
2. 在前端展示并等待用户审批
3. 审批后生成答案草稿
4. Critic 对草稿进行审查
5. 通过后返回答案和引用

这样可以让用户更清楚系统为什么这样回答，也增强了可控性。

### 4. 笔记流程

问答完成后，系统会把最近的对话整理成一份结构化 Markdown 笔记，并自动写入本地文件。笔记更关注“问了什么、得到了什么结论、后续有什么启发”，而不是重新复述整篇论文。

### 5. 前端展示逻辑

前端将页面分为三块：

- 左侧：论文资源管理
- 中间：正文与摘要
- 右侧：问答、计划审批和笔记

同时通过活动时间线展示后端执行过程，让整个系统的处理步骤更直观。

## 运行效果

页面支持：

- 上传论文
- 打开历史资源
- 查看 PDF 正文
- 生成摘要
- 提交论文问题
- 审批答题计划
- 查看答案与引用
- 提取并查看笔记


## 一句话总结

这是一个以 LangGraph 为核心、以 PDF 论文为输入、以摘要 / 问答 / 笔记沉淀为输出的全栈 Demo，重点展示一个可视化、可审批、可复盘的论文助手工作流。
